# دليل المطورين: تطبيق إدارة الخدمات الميدانية  🛠️👨‍💻

هذا المستند مخصص للمطورين لشرح البنية البرمجية، سير العمل التقني، والخطط المستقبلية لتطوير التطبيق.

---

## 🏗️ البنية البرمجية (Architecture)
يعتمد التطبيق على نمط **MVVM** (نموذج - عرض - نموذج العرض) مع استخدام واجهات **Jetpack Compose** الحديثة.

### المكونات الأساسية:
*   **الاتصالات الشبكية:** تعتمد على مكتبة **Retrofit** مع **OkHttp**. تم إعداد "معترضات" (Interceptors) لإضافة رمز المصادقة (JWT) تلقائياً، ومعالجة أخطاء الجلسة (401/403).
*   **نظام التنقل:** يتم عبر **Navigation Component**. تم حل مشكلة "أمان الخيوط" لضمان أن التنقل بين الشاشات يحدث دائماً على الخيط الرئيسي (Main Thread).
*   **إدارة الحالة:** نستخدم `StateFlow` لضمان تحديث البيانات في الواجهة بشكل فوري وتفاعلي.
*   **إدارة الجلسة:** يتم تخزين بيانات المستخدم ورموز المصادقة بشكل آمن عبر كلاس `SessionManager`.

---

## 🔄 سير العمل التقني (User Flows)

### 1. نظام المصادقة والجلسة
*   عند تسجيل الدخول، يتم استلام رمز **JWT**. يُخزن محلياً وتُحدث إعدادات الاتصال فوراً.
*   بعد الدخول، يتم جلب الملف الشخصي لتحديد صلاحيات المستخدم (زبون أو فني).
*   **الأمان:** في حال تم حظر المستخدم من قبل المسؤول، يقوم النظام فوراً بإنهاء الجلسة وإعادة المستخدم لشاشة الدخول.

### 2. تدفق عمليات الزبون
*   **نشر المهام:** يتم إرسال تفاصيل العمل إلى الخادم مع تحديد الموقع والوصف.
*   **إدارة الطلبات:** تم تحسين النظام لعرض بيانات المتقدمين دون أخطاء برمجية في معالجة البيانات (Lazy Initialization).

### 3. تدفق عمليات الفني
*   **تصفح الوظائف:** عرض المهام المتاحة بناءً على تصفية المدن.
*   **إدارة المهام:** واجهة مخصصة لمتابعة المهام المقبولة وتحديث حالتها إلى "مكتملة".

## 🛠️ تفاصيل برمجية هامة

*   **أمان الخيوط (Thread Safety):** تم تجاوز مشكلة الانهيار عند تسجيل الخروج التلقائي عبر ضمان تنفيذ أوامر التنقل ضمن نطاق الخيط الرئيسي.
*   **معالجة الأخطاء:** النظام الآن يفرق بدقة بين "بيانات دخول خاطئة" وبين "حساب محظور" عبر الأكواد البرمجية (401 و 403).
*   **الهوية البصرية:** تم بناء نظام ألوان مخصص (FieldBlack و FieldGold) ليعكس طابعاً احترافياً.

## 🗺️ الخطة المستقبلية: دمج الخرائط و PostGIS

المرحلة القادمة ستركز على استغلال قدرات قاعدة البيانات الجغرافية **PostGIS**:

### 1. على مستوى الخادم (Backend):
*   تفعيل ميزات البحث الجغرافي للعثور على أقرب الفنيين لموقع العمل.
*   تحديث الحقول الجغرافية لتدعم النقاط الإحداثية (Longitude/Latitude).

### 2. على مستوى تطبيق الأندرويد:
*   دمج خرائط جوجل (**Google Maps SDK**).
*   تفعيل ميزة اختيار الموقع عبر الخريطة (Pin Drop) عند نشر الوظائف.
*   إظهار مواقع الفنيين القريبين كنقاط تفاعلية على الخريطة.

## 🛠️ إعداد وتشغيل المشروع

### 1. قاعدة البيانات والخلفية البرمجية (Docker)
يتطلب المشروع قاعدة بيانات PostgreSQL مع إضافة PostGIS. الطريقة الأسهل للتشغيل هي عبر Docker:
*   **تشغيل النظام:**
    ```bash
    docker compose up -d
    ```
*   **تصفير قاعدة البيانات (مسح كافة البيانات):**
    إذا أردت مسح البيانات والبدء من جديد، نفذ الأوامر التالية:
    ```bash
    docker compose down -v
    docker compose up -d
    ```

### 2. تطبيق الأندرويد (USB Debugging)
لتشغيل التطبيق على هاتف حقيقي عبر وصلة USB:
1.  **تفعيل خيارات المطور:** اذهب إلى `Settings > About Phone` واضغط على `Build Number` 7 مرات متتالية.
2.  **تفعيل تصحيح USB:** من `Developer Options` قم بتفعيل `USB Debugging`.
3.  **ربط المنافذ عبر ADB:**
    بما أن التطبيق يتصل بـ `localhost:8080` على جهاز الكمبيوتر، يجب تنفيذ هذا الأمر في سطر الأوامر (Terminal) لربط منفذ الهاتف بجهازك:
    ```bash
    adb reverse tcp:8080 tcp:8080
    ```
    *(ملاحظة: تأكد من إضافة مسار أدوات `adb` إلى متغيرات النظام PATH)*.

### 3. لوحة تحكم المسؤول (Web)
لتشغيل لوحة تحكم الويب:
1.  افتح مجلد `fsm-frontend` باستخدام برنامج VS Code.
2.  افتح سطر الأوامر ونفذ الأمر: `npm install` (لأول مرة فقط).
3.  شغل مشروع الويب عبر الأمر:
    ```bash
    npm run dev
    ```
